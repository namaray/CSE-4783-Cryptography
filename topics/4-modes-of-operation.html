1. Overview & Technical Notes

Block ciphers (like DES) encrypt fixed-size blocks (e.g., 64-bit blocks). To use them in practice for arbitrary amounts of information, we partition the message into separate blocks. A Mode of Operation describes the process of encrypting each of these blocks under a single key.

Key Concepts

  • Initialize Vector (IV): A block of bits used to randomize the encryption, ensuring that the same plaintext produces distinct ciphertexts.
  • Nonce (Number Used Once): A random or pseudo-random number ensuring past communications cannot be reused in replay attacks. (Sometimes used interchangeably with IV).
  • Padding: The final block may require padding to fit the fixed block size. Methods include adding null bytes, adding 0x80 followed by 0x00s, or adding n bytes with the value n (PKCS padding).

2. Electronic Codebook (ECB) Mode

The message is broken into independent blocks which are encrypted separately. Each block is a value substituted like a codebook.

Encryption: \( C_i = E_K(P_i) \)
  • Strengths: It's very simple.
  • Weaknesses: Repetitive information in the plaintext will show up as identical blocks in the ciphertext. If the same message is sent twice, the ciphertext is identical.
  • Typical Application: Secure transmission of short pieces of single values (e.g., sending a temporary encryption key).

3. Cipher Block Chaining (CBC) Mode

Solves the security deficiencies of ECB. Each previous cipher block is chained as an input with the current plaintext block before encryption.

\( C_0 = IV \)
Encryption: \( C_i = E_K(P_i \oplus C_{i-1}) \)
  • Repeated plaintext blocks are encrypted differently because the encryption of a block depends on the current and all blocks before it.
  • Reusing an IV leaks some information about the first block of plaintext, and about any common prefix shared by two messages.
  • Typical Application: Bulk data encryption, Authentication.

4. CFB and OFB Modes (Stream Ciphers)

Both of these modes turn a block cipher into a stream cipher, allowing you to encrypt any number of bits (like single bits or bytes) at a time.

Cipher FeedBack (CFB)

Encrypts the previous ciphertext, then combines it with the plaintext block using XOR.

\( C_0 = IV \)
Encryption: \( C_i = P_i \oplus \text{SelectLeft}(E_K(\text{ShiftLeft}(C_{i-1}))) \)
  • A corrupted ciphertext segment during transmission will affect the current and next several plaintext segments.
  • Typical Application: Stream data encryption, Authentication.

Output FeedBack (OFB)

Similar to CFB, but the output of the encryption function is fed back, instead of the ciphertext. The feedback is independent of the message.

\( O_0 = IV \)
Key Stream: \( O_i = E_K(O_{i-1}) \)
Encryption: \( C_i = P_i \oplus O_i \)
  • Each bit in the ciphertext is independent of the previous bits, which avoids error propagation over noisy channels.
  • Pre-computing the forward cipher is possible.
  • Security Issue: If plaintext is known, the key stream \( O_i \) is known, which can easily decrypt other messages using the same IV. Reusing an IV completely destroys security.
  • Typical Application: Stream encryption over noisy channels.

5. Counter (CTR) Mode

Encrypts a counter value with the key, rather than using any feedback. The counter for each plaintext block must be different and unpredictable.

Key Stream: \( O_i = E_K(Counter_i) \)
Encryption: \( C_i = P_i \oplus O_i \)
  • Strengths: Fast encryption/decryption, allows parallel processing (because there is no chaining/dependency on previous blocks), and provides random access to encrypted data blocks.
  • Needs only the encryption algorithm (decryption just generates the same key stream and XORs it).
  • Typical Application: General purpose block-oriented transmission, High-speed network encryptions.

6. Comparison of Modes

CBC vs CTR

Feature CBC (Cipher Block Chaining) CTR (Counter)
Padding Padding is needed No padding needed
Processing No parallel processing Parallel processing allowed
Functions Needed Separate Encryption and Decryption functions Encryption function alone is enough
Initialization Random IV or Nonce Unique Nonce
IV Reuse Penalty Leaks some info about initial plaintext block Leaks info about the entire message

Summary Table

Mode Type Application
ECB Block Cipher Secure transmission of a single encryption key.
CBC Block Cipher Commonly used method. Used for authentication and bulk data.
CFB Stream Cipher Primary stream cipher. Used for authentication.
OFB Stream Cipher Stream cipher well suited for transmission over noisy channels.
CTR Stream Cipher General purpose. Used for high-speed communications.
