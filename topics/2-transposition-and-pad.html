1. Transposition Ciphers

A transposition cipher reorders (transposes) the symbols in the plaintext, thereby creating the ciphertext. It does not change the actual characters, only their positions.

Keyless Transposition Ciphers

Traditional and easy to use. The plaintext characters are written into a table either column-wise or row-wise. Ciphertext is formed by reading the characters in the opposite manner (e.g., write horizontally, read vertically).

  • Staff Cipher: Earliest version used by the Spartans in the 5th century BC.
  • The secrecy relies entirely on the width of the table.
  • If the message doesn't perfectly fit the table, the empty spaces are padded out with dummy characters (like x, y, z).

Keyed Transposition Ciphers

Divides the plaintext into blocks of predefined size. A key is used to permute the characters within each block.

Example: Enter data horizontally in a table. Read down columns, but sort the columns based on the numbered key order.

Key: 3 1 4 2 M Y C A T H A S F L E A S x y z Read column 1 (key order 1): YHLx Read column 2 (key order 2): ASAz Read column 3 (key order 3): MTFS Read column 4 (key order 4): CAEy

Note: A transposition cipher is actually a special case of a Hill cipher because permutations can be performed using matrix multiplication. It does nothing to hide the statistical distribution of letters.

2. Stream Ciphers & One Time Pad

One Time Pad (Vernam Cipher)

First implemented at AT&T using the Vernam machine. It uses a random set of non-repeating characters as a key. The key must be used only once.

  1. Assign alphabets in plaintext to numbers (A=0, B=1 ... Z=25).
  2. Assign randomly chosen pad characters to numbers.
  3. Add the plaintext numbers to the pad numbers.
  4. Apply Mod 26 operation to the sum.
  5. Translate back to alphabets to get the ciphertext.

Binary One Time Pad

In modern computing, the plaintext \( M \) and pad \( P \) are viewed as random binary strings of length \( n \). Encryption and decryption are performed using the bitwise XOR operator (\( \oplus \)).

Encryption: \( C = M \oplus P \)
Decryption: \( M = C \oplus P \)

Security: The binary one-time pad is absolutely unbreakable. Each bit of ciphertext is equally likely to be a 0 or 1, independent of the plaintext.

3. Modern Block Ciphers

Earlier ciphers were character-oriented. Modern ciphers are bit-oriented. This is because modern data includes graphics, audio, and video, which must be transformed into streams of bits. Replacing characters with 8-bit or 16-bit blocks increases security exponentially.

Block Ciphers vs Stream Ciphers

  • Block Cipher: Encrypts an \( m \)-bit block of plaintext at a time to produce an \( m \)-bit block of ciphertext using a fixed key.
  • Partitioning: Plaintext of length \( n \) is partitioned into a sequence of blocks.
  • Padding: Required when plaintext length is not a multiple of the block size. A common method is PKCS5.

Padding Example: If block size is 16 bytes, and plaintext is "Roberto" (7 bytes). It needs 9 bytes of padding. Padded text: Roberto999999999 (where 9 denotes the length of the padding bytes).

4. Components of Modern Block Ciphers

S-Box (Substitution Box)

Takes \( n \) bits of input and produces \( m \) bits of output. Substitution of several bits is performed in parallel. S-boxes can be keyed or keyless (keyless are common in modern ciphers).

P-Box (Permutation / Diffusion Box)

Performs transposition at the bit-level. Several bits are permuted simultaneously.

  • Straight P-box: \( n \) inputs to \( n \) outputs. There are \( n! \) possible mappings.
  • Compression P-box: \( n \) inputs to \( m \) outputs (where \( m < n \)). Some inputs are blocked and dropped.
  • Expansion P-box: \( n \) inputs to \( m \) outputs (where \( m > n \)). A single input is mapped to more than one output.

Circular Shift

Bits are shifted either left or right to conceal bit patterns. In a circular left shift by \( n \) positions, the \( n \) leftmost bits are removed and placed at the rightmost positions.

5. Shannon's Theory: Diffusion and Confusion

Proposed by Claude Shannon to thwart cryptanalysis based on statistical analysis.

  • Diffusion: Hides the relationship between ciphertext and plaintext. Frustrates statistical analysis. A change in a single plaintext symbol causes changes in several (or all) ciphertext symbols. (Achieved via P-Boxes).
  • Confusion: Hides the relationship between ciphertext and the key. Frustrates attempts to discover the key. A change in a single bit of the key causes changes in several ciphertext symbols. (Achieved via S-Boxes).
