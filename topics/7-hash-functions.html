One-way functions for integrity verification and authentication

1. What is a Cryptographic Hash Function?

A hash function \( H \) takes an input of arbitrary length and produces an output (the digest or hash value) of fixed length. It is a one-way, deterministic function.

\( h = H(M) \)
where \( M \) = message of any length, and \( h \) = fixed-size digest (e.g., 256 bits for SHA-256)

Hash functions are not used for encryption (you cannot decrypt a hash). They are used for data integrity verification.

2. Three Critical Security Properties

🔒

Pre-image Resistance

Given \( h \), it is computationally infeasible to find any \( M \) such that \( H(M) = h \).
(One-Way)

🔑

Second Pre-image Resistance

Given \( M_1 \), it is infeasible to find \( M_2 \neq M_1 \) such that \( H(M_1) = H(M_2) \).
(Weak Collision Resistance)

💥

Collision Resistance

It is infeasible to find any two inputs \( M_1 \neq M_2 \) that produce the same hash: \( H(M_1) = H(M_2) \).
(Strong Collision Resistance)

💡 Hierarchy: Collision Resistance implies Second Pre-image Resistance. If you can't find any collision, you certainly can't find one for a specific \( M_1 \).

3. The Birthday Paradox & Attack

The Birthday Attack exploits the Birthday Paradox from probability theory to find hash collisions much faster than expected.

The Birthday Paradox

In a group of just 23 people, there is a >50% chance that two people share a birthday. This is because we're looking for any matching pair, not a match to a specific birthday.

Application to Hash Functions

For a hash function with an \( n \)-bit output, there are \( 2^n \) possible digests. A brute-force pre-image attack requires \( 2^n \) attempts. But a collision attack only requires approximately:

Collision Attack Complexity: \( \approx 2^{n/2} \) attempts

For SHA-256 (\( n = 256 \)): \( 2^{256/2} = 2^{128} \) attempts (still computationally infeasible)
🚨 MD5 and SHA-1 are BROKEN! MD5 (128-bit) is vulnerable to collision attacks (only \( 2^{64} \) effective, and practical collisions have been found). SHA-1 (160-bit) collision was demonstrated by Google in 2017 (SHAttered attack). Neither should be used for security-sensitive applications.

5. Merkle-Damgård Construction

The design used by MD5, SHA-1, and SHA-2 (SHA-256). It processes the message in fixed-size blocks using an iterative compression function.

  1. Padding: The message is padded to a multiple of the block size. A "1" bit is appended, followed by "0" bits, followed by the 64-bit length of the original message.
  2. Initialization: A fixed initial hash value \( H_0 \) (IV) is defined.
  3. Compression: For each message block \( M_i \): \( H_i = f(H_{i-1}, M_i) \) where \( f \) is the compression function.
  4. Output: The final hash \( H_n \) is the digest.
\( H_i = f(H_{i-1}, M_i) \)
Final digest = \( H_n \)

6. Applications of Hash Functions

  • Data Integrity: Download a file and verify its hash matches the published hash to ensure it wasn't corrupted/tampered.
  • Password Storage: Store \( H(\text{password}) \) instead of the plaintext password. Use salting (add a random value before hashing) to prevent rainbow table attacks.
  • Digital Signatures: Sign \( H(M) \) instead of \( M \) directly (more efficient, especially for large documents). Covered in Module 8.
  • HMAC (Hash-based Message Authentication Code): Combines a secret key with a hash to produce an authentication tag, providing both integrity and authentication.
  • Blockchain: Each block contains the hash of the previous block, creating a tamper-evident chain.
  • Pseudorandom Number Generation (PRNGs): Hash functions can be used to build cryptographically secure PRNGs.
HMAC: \( \text{HMAC}(K, M) = H\big( (K \oplus \text{opad}) \| H((K \oplus \text{ipad}) \| M) \big) \)
where opad and ipad are fixed padding constants.

7. SHA-256 Internal Structure (High-Level)

SHA-256 operates on 512-bit message blocks and uses an internal state of 256 bits (8 × 32-bit words: A, B, C, D, E, F, G, H).

  • Message Schedule: Each 512-bit block is expanded into 64 × 32-bit words \( W_0 ... W_{63} \).
  • Compression Rounds: 64 rounds of mixing using bitwise operations (AND, XOR, NOT, ROTR), modular addition, and 64 pre-defined constants \( K_i \) (first 32 bits of the fractional parts of cube roots of the first 64 primes).
  • Sigma Functions: Non-linear mixing uses specific right-rotation functions: \( \Sigma_0, \Sigma_1, \sigma_0, \sigma_1 \).
  • Output: The 8 words are concatenated to form the 256-bit digest.
