1. What is Steganography?

Steganography hides the existence of a secret message, unlike encryption which hides only its content. The goal is that even a suspicious observer cannot detect that any message has been hidden.

FeatureEncryptionSteganography
HidesThe content of a messageThe existence of a message
Detectable?Yes — encrypted data is visibleNo — cover medium looks innocent
If key is foundMessage is fully exposedStill needs stego key to extract

Key Terminology

  • Cover Image (I): The original, unmodified carrier image.
  • Stego Image (S): The image after the secret message is embedded. Must look visually identical to I.
  • Payload: The total number of bits embedded in the image.
  • Embedding Capacity (bpp): Bits Per Pixel — how many bits can be hidden per pixel on average.

Choice of Cover Media

Different types of digital files can be used as the cover medium, but they perform differently for steganography:

Media TypeCharacteristics for Steganography
TextHas very little redundant data. Any alterations or distortions are easily realizable (visible) to the reader.
Audio / VideoBigger file size offers massive capacity, but makes it less flexible and heavier for frequent communication/transmission.
Image (Most Popular)
  • Has abundant redundant information to hide data.
  • Suitable size for frequent transmission over networks.
  • Big enough to hold a meaningful amount of embedded data.
  • Smaller distortions (like modifying LSBs or small errors) are not sensible to the human visual system.

2. Domains of Image Steganography

Embedding can happen in three different domains:

DomainWhat is ModifiedExample Technique
Image DomainRaw pixel values directlyLSB Replacement
Transform DomainFFT / Wavelet / DCT coefficientsJPEG steganography
Prediction ErrorsErrors between predicted and actual pixel values — the histogram of these errors is shifted to embed dataPrediction Error Histogram (PEH) — the main focus
💡 Why Prediction Errors? Natural images are smooth — neighbouring pixels are highly correlated. A predictor can estimate each pixel very accurately, leaving only small errors. These small, concentrated errors form a sharp histogram that is ideal for reversible steganography.

3. Performance Measuring Parameters

Two metrics measure the quality of the embedding. Cover image is \( I \), stego image is \( S \), dimensions \( w \times h \).

Volume of Embedded Bits

\( \text{Payload} = \text{Total embedded bits} \)

\( \text{Embedding Capacity (bpp)} = \dfrac{\text{Payload}}{w \times h} \)

Stego Image Quality

\( \text{MSE} = \dfrac{\displaystyle\sum_{i}\sum_{j}(I_{i,j} - S_{i,j})^2}{w \times h} \)

\( \text{PSNR} = 10\log_{10}\!\left(\dfrac{255^2}{\text{MSE}}\right) \text{ dB} \)
📌 Interpretation: Higher PSNR = better quality (less distortion). A PSNR above 40 dB is typically considered visually lossless. Lower MSE is better.

4. Steganography Classification

Classified by whether the original cover image can be recovered after message extraction:

TypeCover Recoverable?Techniques
Irreversible (Lossy)❌ No — permanently alteredLSB Replacement (basic)
Reversible (Lossless)✅ Yes — original can be perfectly restored after extractionBasic Pixel Reference Error, Prediction Error Histogram
💡 Why Reversible? Critical for medical imaging, military, or legal documents where the original image must be perfectly preserved after the hidden message is read.

5. Pixel Prediction

Before computing errors, a predictor estimates each pixel's value from its neighbours. Two approaches from the slides:

Method 1: Single Pixel Prediction (Sachnev et al., 2009)

Each pixel \( I_i \) is predicted as a weighted average of its 4 neighbours (Top, Right, Bottom, Left):

\( P_i = \sum_{j \in \{\text{Top, Right, Bottom, Left}\}} w_j \cdot I_j \)

Weights \( w_j \) are chosen to minimise prediction error for the image type (e.g., \( w = 0.25 \) for each neighbour gives a simple average).

Method 2: Block Pixel Prediction (Hong W. et al., 2010)

Instead of relying on adjacent pixels, the image is divided into non-overlapping blocks (e.g., 3×3). Within each block, the centre pixel \( c_i \) acts as the primary reference to predict its surrounding 8 satellite pixels \( p_{i,j} \). The prediction also uses the centre pixels of adjacent blocks (\( c_i^L, c_i^T, c_i^R, c_i^B \)).

\( p_{i,1} = \text{round}(\frac{1}{3}(2c_i + c_i^L)) \)
\( p_{i,2} = \text{round}(\frac{1}{3}(c_i + c_i^L + c_i^T)) \)
\( p_{i,3} = \text{round}(\frac{1}{3}(2c_i + c_i^T)) \)
\( p_{i,4} = \text{round}(\frac{1}{3}(c_i + c_i^T + c_i^R)) \)
\( p_{i,5} = \text{round}(\frac{1}{3}(2c_i + c_i^R)) \)
\( p_{i,6} = \text{round}(\frac{1}{3}(c_i + c_i^R + c_i^B)) \)
\( p_{i,7} = \text{round}(\frac{1}{3}(2c_i + c_i^B)) \)
\( p_{i,8} = \text{round}(\frac{1}{3}(c_i + c_i^B + c_i^L)) \)

Prediction Error: \( e_{i,j} = I_{i,j} - p_{i,j} \)    for \( j \in \{1, 2, \dots, 8\} \)

The centre pixel \( c_i \) remains unchanged during the entire embedding process. By predicting satellite pixels using a weighted average of the centre pixel and adjacent block centre pixels, this method allows up to 8 bits to be embedded per 3×3 block while maintaining high reversibility.

6. Prediction Error Calculation

Once predicted values \( P_{i,j} \) are computed, the Prediction Error \( E_{i,j} \) for each pixel is:

\( E_{i,j} = I_{i,j} - P_{i,j} \)

These errors form the embedding space — the secret message bits will be hidden inside the error values.

📌 Key Property: For smooth natural images, most errors cluster tightly around 0 — the predictor is highly accurate. This creates a sharp, peaked histogram — ideal for the PEH method.

7. The Error Histogram

Plot the frequency of each error value across all pixels. Because the predictor is accurate, the histogram forms a sharp peak near zero.

Key Histogram Values (Notation)

SymbolMeaning
\( P_+ \)The positive error value with the highest frequency (positive peak)
\( P_- \)The negative error value with the highest frequency (negative peak)
\( Z_+ \)The highest positive error value present in the histogram
\( Z_- \)The highest negative error value (most negative) present
\( h_1 \)Frequency count at \( P_- \)
\( h_2 \)Frequency count at \( P_+ \)
💡 Embedding capacity ≈ \( h_1 + h_2 \) — we can embed one bit per pixel at each of the two peak positions.

8. Embedding on the Error Histogram

Three operations are applied to each error value \( r_{i,j} \) to embed a message bit \( b \in \{0,1\} \):

Step 1 — Shift non-embeddable errors outward (make room)

If \( r_{i,j} > P_+ \):    \( \tilde{r}_{i,j} = r_{i,j} + 1 \)
If \( r_{i,j} < P_- \):    \( \tilde{r}_{i,j} = r_{i,j} - 1 \)

This frees up the slots just above \( P_+ \) and just below \( P_- \) for data embedding.

Step 2 — Embed at the positive peak \( P_+ \)

If \( r_{i,j} = P_+ \):    \( \tilde{r}_{i,j} = r_{i,j} + b \)

→ If \( b = 0 \): error stays at \( P_+ \) (no change)
→ If \( b = 1 \): error moves to \( P_+ + 1 \)

Step 3 — Embed at the negative peak \( P_- \)

If \( r_{i,j} = P_- \):    \( \tilde{r}_{i,j} = r_{i,j} - b \)

→ If \( b = 0 \): error stays at \( P_- \) (no change)
→ If \( b = 1 \): error moves to \( P_- - 1 \)

Step 4 — Reconstruct the Stego Image

\( \tilde{b}_{i,j} = p_{i,j} + \tilde{e}_{i,j} \)

Add the modified error back onto the predicted value to get the stego pixel value. Embedding is complete!

9. Message Extraction at the Receiver

The receiver recomputes the prediction \( \tilde{p}_{i,j} \) from the stego image (the centre/reference pixels are unchanged), then recovers the stego errors \( \tilde{e}_{i,j} = \tilde{b}_{i,j} - \tilde{p}_{i,j} \).

Extract the bit \( s \)

\( s = 0 \) if \( \tilde{e}_{i,j} = P_+ \)  or  \( \tilde{e}_{i,j} = P_- \)

\( s = 1 \) if \( \tilde{e}_{i,j} = P_+ + 1 \)  or  \( \tilde{e}_{i,j} = P_- - 1 \)

Reconstruct the original error (reversibility)

\( e_{i,j} = \tilde{e}_{i,j} - 1 \)   if \( \tilde{e}_{i,j} > P_+ \)
\( e_{i,j} = \tilde{e}_{i,j} + 1 \)   if \( \tilde{e}_{i,j} < P_- \)
\( e_{i,j} = \tilde{e}_{i,j} \)        otherwise

The original pixel is then recovered as \( I_{i,j} = P_{i,j} + e_{i,j} \). Extraction is done — the cover image is fully restored.

11. Critics on the PEH Scheme

Critic 1: Distortion of visual quality greatly depends on data bits. More '1' bits in the message means more pixels are shifted — more distortion.
Critic 2: The strategy cannot embed to all pixels. A variance control mechanism skips blocks with large variance (less smooth blocks), which reduces payload.
Critic 3: PEH selects two highest peaks — one positive \( P_+ \), one negative \( P_- \). But experiments show that at some images, both highest peaks are on the positive side, making the two-peak selection suboptimal.
Critic 4: Boundary blocks lack some satellite pixels — boundary blocks must be avoided, which reduces total capacity.
Critic 5: Embedding to only two peaks limits payload to less than 20% of image data. Payload remains a fundamental limitation.

12. Multi-Layer Embedding

To increase capacity beyond what a single peak pair allows, Multi-Layer Embedding repeats the process across multiple error layers.

  • Layer 0: Embed into errors at the two primary peaks (standard PEH).
  • Layer 1: Embed into errors at positions ±1 from the primary peaks.
  • Layer 2, 3…: Continue expanding the embeddable range outward.

Embedding Layer \( L \) — Bit Mapping

When error \( e \geq 0 \):
   If \( b = 0 \): \( \tilde{e} = 2e \)
   If \( b = 1 \): \( \tilde{e} = 2e + 1 \)

When error \( e < 0 \):
   If \( b = 0 \): \( \tilde{e} = 2e + 1 \)
   If \( b = 1 \): \( \tilde{e} = 2e \)

Non-embeddable errors (outside layer \( L \)) are shifted by \( L \) places outward. This significantly increases payload at the cost of slightly greater distortion per embedded bit.

💡 Trade-off: More layers = higher capacity but also higher distortion (lower PSNR). Optimum layer count is chosen based on the required payload and acceptable PSNR.
