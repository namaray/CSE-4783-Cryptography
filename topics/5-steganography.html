1. Steganography vs. Cryptography

While Cryptography scrambles a message so it cannot be read, Steganography hides the message so it cannot even be detected. Both are part of the broader field of information hiding.

🔐 Cryptography

  • Hides the content of a message
  • The existence of communication is visible
  • Adversary knows something is hidden but can't read it
  • Example: Encrypted email traffic

🕵️ Steganography

  • Hides the existence of the message
  • The cover medium looks completely innocent
  • Adversary does not suspect any communication
  • Example: Hidden text in an image file
💡 Key Insight: Steganography and Cryptography can be combined — first encrypt the secret message, then hide the ciphertext using steganography. This provides both secrecy and concealment.

2. Core Components

  • Cover Medium (C): The innocent carrier that hides the secret. Can be an image, audio file, video, or text document.
  • Secret Message (M): The hidden data to be concealed.
  • Stego-Key (K): An optional key used to embed/extract the message, adding another layer of security.
  • Stego-Object (S): The cover medium after the secret message has been embedded. It should be perceptually identical to the original cover.
Embedding: \( S = \text{Embed}(C, M, K) \)
Extraction: \( M = \text{Extract}(S, K) \)

3. Image Steganography: LSB Technique

The most common digital steganography technique. Each pixel in an image has color values (e.g., RGB). Each value is stored as an 8-bit number. The Least Significant Bit (LSB) of each byte contributes the smallest change in color — a change that is imperceptible to the human eye.

How LSB Works

By replacing the LSB of each pixel's color value with a bit of the secret message, we can hide data with minimal visual distortion.

Original Pixel (R, G, B)Binary RepresentationAfter Embedding bit '1'
R = 2001100 10001100 1001 = 201
G = 1301000 00101000 0011 = 131
B = 750100 10110100 1011 = 75 (no change)

A maximum color change of 1 out of 255 is visually undetectable.

Storage Capacity

For a 24-bit (RGB) image of size \( W \times H \) pixels with 1 bit per channel:
Max hidden data = \( W \times H \times 3 \) bits = \( \frac{W \times H \times 3}{8} \) bytes
⚠️ Trade-off: Using more bits per pixel (e.g., 4 LSBs) increases capacity but also increases visual distortion, making detection easier.

4. Other Steganography Techniques

Text Steganography

  • Whitespace Manipulation: Extra spaces or tabs are added at the end of lines to encode bits.
  • Line Shifting: Text lines are shifted vertically by small amounts.
  • Word Shifting: Words are shifted horizontally.
  • Syntactic Methods: Using synonyms or changing punctuation to encode data.

Audio Steganography

  • LSB Coding: Same concept as image LSB applied to audio sample bytes.
  • Echo Hiding: Embeds information by adding an echo signal to the audio. The echo's delay, amplitude, and initial value represent data.
  • Phase Coding: Alters the phase components of audio segments to encode data.

Network / Protocol Steganography

  • Hiding data in unused header fields of network packets (e.g., IP, TCP, HTTP headers).
  • Manipulating inter-packet timing to encode information.

5. Steganalysis (Detection)

Just as cryptanalysis attempts to break ciphers, Steganalysis attempts to detect or decode hidden messages. The goal is to distinguish between clean cover media and stego-objects.

  • Statistical Analysis: LSB steganography slightly alters the statistical distribution of pixel values. Tools can detect this deviation from the expected distribution of a natural image.
  • Visual Attack: Extracting and visualizing the LSBs of an image. Hidden data appears as random noise, while natural image LSBs show structure.
  • Chi-Square Attack: Compares the expected frequency distribution of pixel pairs in a cover image versus a stego-image.
💡 Countermeasure: Spread the hidden data randomly throughout the image (using a pseudo-random number generator seeded with a key) rather than sequentially to resist statistical attacks.

6. Key Properties of a Good Steganographic System

PropertyDescription
ImperceptibilityThe stego-object must be indistinguishable from the cover to the human senses.
CapacityThe amount of data that can be hidden without causing perceptible distortion.
RobustnessThe hidden data should survive common signal processing operations (compression, cropping).
SecurityA steganalyst should not be able to prove that a stego-object contains hidden data.
⚠️ The Steganography Triangle: Imperceptibility, Capacity, and Robustness cannot all be maximized simultaneously. Increasing one property generally degrades another.
