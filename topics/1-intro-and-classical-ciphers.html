1. Core Terminology & Concepts

Cryptography (from kryptos: secret writing) is the process of altering messages to hide their meaning from adversaries.

  • Plaintext (P/M): The original, unencrypted message.
  • Ciphertext (C): The encrypted message received by the receiver.
  • Encryption (E): Transforming plaintext into ciphertext. \( C = E(M) \)
  • Decryption (D): Transforming ciphertext back to plaintext. \( M = D(C) \)
  • Key (K): A number/set of numbers the cipher operates on.

The 7 Elements of a Cryptosystem:

  1. Set of possible plaintexts
  2. Set of possible ciphertexts
  3. Set of encryption keys
  4. Set of decryption keys
  5. Correspondence between encryption and decryption keys
  6. The encryption algorithm
  7. The decryption algorithm

2. Symmetric vs Public-Key Cryptography

Symmetric Key Cryptography Public Key (Asymmetric) Cryptography
Uses a single key for both encryption and decryption. Uses two different keys (Public for encryption, Private for decryption).
Requires \( \frac{n(n-1)}{2} \) keys for n users communicating in a network. Key distribution is not a major problem.
Very fast processing. Slower compared to symmetric-key.
Used only for confidentiality. Used for confidentiality, integrity, and digital signatures.
Examples: DES, AES. Example: RSA.

3. Cryptanalysis & Key Management

Types of Cryptanalysis Attacks:

  • Ciphertext-only attack: Attacker only has the ciphertext.
  • Known-plaintext attack: Attacker has previous pairs of (plaintext, ciphertext).
  • Chosen-plaintext attack: Attacker can choose plaintexts and get their ciphertexts.
  • Chosen-ciphertext attack: Attacker can choose ciphertexts and get their decrypted plaintexts.

Key Management Functions:

Generation: Must be random and long enough to avoid prediction.
Distribution: Moving the key securely from generation point to usage point.
Installation: Securely storing it on the device.
Storage: Maintaining confidentiality of stored keys.
Change: Keys must be changed periodically to prevent interception.
Control: Implementing restraining influence over key usage.

4. Substitution Ciphers (Math & Formulas)

Substitution ciphers replace symbols in the plaintext with other symbols. They are divided into Monoalphabetic (1-to-1 relationship) and Polyalphabetic (1-to-many relationship).

Additive / Shift / Caesar Cipher

Encryption: \( C = (P + K) \pmod{26} \)
Decryption: \( P = (C - K) \pmod{26} \)

Example: Key = 20. Plaintext 't' (value 19). \( C = (19 + 20) \pmod{26} = 39 \pmod{26} = 13 \) ('n').

Multiplicative Cipher

Encryption: \( C = (P \times K) \pmod{26} \)
Decryption: \( P = (C \times K^{-1}) \pmod{26} \)

Example: Key = 15. The modular inverse of 15 modulo 26 is 7 (because \( 15 \times 7 = 105 \equiv 1 \pmod{26} \)). To decrypt, multiply the ciphertext by 7.

Affine Cipher

Combines Additive and Multiplicative ciphers using a pair of keys \( (K_1, K_2) \).

Encryption: \( C_1 = (P \times K_1) \pmod{26} \), then \( C = (C_1 + K_2) \pmod{26} \)
Decryption: \( P_1 = (C - K_2) \pmod{26} \), then \( P = (P_1 \times K_1^{-1}) \pmod{26} \)

Vigenère Cipher (Polyalphabetic)

The key is repeated until it matches the length of the plaintext.

\( C_i = (P_i + K_i) \pmod{26} \)

5. Playfair Cipher

Uses a 5x5 matrix based on a keyword (I and J are combined). Plaintext is split into pairs (digraphs). If a pair has the same letters, insert 'X'.

Encryption Rules:

  1. Same Row: Replace each letter with the letter to its right (wrap around).
  2. Same Column: Replace each letter with the letter below it (wrap around).
  3. Rectangle Rule: Replace each letter with the letter in its same row, but the other column of the rectangle.

Note: For decryption, reverse the rules (Move Left for rows, Move Up for columns).

6. Hill Cipher (Matrix Math)

The Hill cipher encrypts blocks of text using matrix multiplication. The key \( K \) is an \( n \times n \) matrix.

Encryption: \( C = K \times P \pmod{26} \)
Decryption: \( P = K^{-1} \times C \pmod{26} \)

Steps to find the Inverse Key Matrix \( K^{-1} \) for Decryption:

  1. Compute the Determinant: \( det(K) \)
  2. Find the Inverse Determinant: Find \( x \) such that \( det(K) \times x \equiv 1 \pmod{26} \)
  3. Compute the Cofactor Matrix: Calculate the cofactor for every position in the matrix.
  4. Compute the Adjugate Matrix: Transpose the Cofactor matrix.
  5. Multiply: Multiply the Adjugate matrix by the Inverse Determinant modulo 26.

Example from slides: Key = "GYBNQKURP" translates to a 3x3 matrix. Determinant calculated is 441. \( 441 \equiv 25 \pmod{26} \). The modular inverse of 25 mod 26 is 25. The adjugate matrix is multiplied by 25 mod 26 to get \( K^{-1} \).
