1. Core Terminology & Concepts

Cryptography (from kryptos: secret writing) is the process of altering messages to hide their meaning from adversaries. It is essential for communicating over untrusted mediums like the Internet.

  • Plaintext (P/M): The original unencrypted message.
  • Ciphertext (C): The encrypted message received by the receiver.
  • Encryption (E): The process of producing ciphertext. \( C = E(M) \)
  • Decryption (D): The reverse process; converting ciphertext back to plaintext. \( M = D(C) \)
  • Key (K): A number or set of numbers on which the cipher operates. The longer the key, the harder it is for an attacker.

The 7 Elements of a Cryptosystem:

  1. Set of possible plaintexts
  2. Set of possible ciphertexts
  3. Set of encryption keys
  4. Set of decryption keys
  5. Correspondence between encryption and decryption keys
  6. The encryption algorithm to use
  7. The decryption algorithm to use

2. Symmetric vs Public-Key Cryptography

Symmetric Key Cryptography Public Key (Asymmetric) Cryptography
Uses a single key for both encryption and decryption. Uses two different keys (Public for encryption, Private for decryption).
Key distribution is a big problem. Requires \( \frac{n(n-1)}{2} \) keys for n users. Key distribution is not a problem.
Very fast processing. Ciphertext size usually ≤ original text. Slower processing. Ciphertext size usually > original text.
Used only for confidentiality. Used for confidentiality, integrity, and non-repudiation (Digital Signatures).
Algorithms: DES, AES. Algorithm: RSA.

3. Cryptanalysis Attacks

Cryptanalysis is the art of breaking encrypted codes to obtain the plaintext or key. There are four primary models based on the attacker's access:

1. Cyphertext-Only Attack

Only Intercepts Ciphertext

The attacker analyze the intercepted ciphertext alone to find the plaintext or key.

2. Known-Plaintext Attack

Has Previous P/C Pairs

Attacker uses knowledge of previous plaintext-ciphertext pairs to help break the current message.

3. Chosen-Plaintext Attack

Access to Encryption Process

Attacker chooses specific plaintexts to be encrypted and analyzes the resulting ciphertexts.

4. Chosen-Ciphertext Attack

Access to Decryption Process

Attacker chooses specific ciphertexts to be decrypted and analyzes the resulting plaintexts.

4. Symmetric Key Ciphers

Broadly divided into Substitution (changing character identity) and Transposition (changing character position).

Monoalphabetic Ciphers (1-to-1 relationship)

Additive / Shift / Caesar Cipher: Shifts every letter by a fixed key \(K\).

\( C = (P + K) \pmod{26} \) | \( P = (C - K) \pmod{26} \)

Multiplicative Cipher: Multiplies value by key \(K\). Requires \(gcd(K, 26) = 1\).

\( C = (P \times K) \pmod{26} \) | \( P = (C \times K^{-1}) \pmod{26} \)

Affine Cipher: A combination of both. Uses key pair \((K_1, K_2)\).

\( C = (P \times K_1 + K_2) \pmod{26} \) | \( P = (C - K_2) \times K_1^{-1} \pmod{26} \)

Polyalphabetic Ciphers (1-to-many relationship)

  • Autokey Cipher: The key stream is generated from a seed followed by the plaintext itself.
  • Vigenère Cipher: Uses a keyword that repeats. Each letter of the keyword determines the shift for that position.
  • Hill Cipher: A mathematical cipher treating blocks of letters as vectors and multiplying by an \(n \times n\) matrix.

5. Advanced Classical Logic

Playfair Preparation Rules

Before encrypting digraphs (pairs), process the plaintext:

  • If a pair has the same letter, insert an 'X' (e.g., HELLO → HE LX LO).
  • If one letter remains at the end, add an 'X'.

Hill Cipher Matrix Math

\( C = K \cdot P \pmod{26} \) | \( P = K^{-1} \cdot C \pmod{26} \)
  1. Compute Determinant \(det(K)\).
  2. Find Modular Inverse of determinant s.t. \(det(K) \cdot x \equiv 1 \pmod{26}\).
  3. Find Cofactor Matrix and transpose it to get Adjugate Matrix.
  4. Multiply Adjugate Matrix by inverse determinant \(\pmod{26}\) to get \(K^{-1}\).

6. Transposition & Modern Transition

Transposition Ciphers

Keyless (Staff Cipher): Enter data horizontally in a table, read it vertically. Width of the table determines secrecy.

Keyed: Divide plaintext into blocks and use a key to reorder (permute) the characters in each block individually.

Transition to Modern Ciphers

  • Character-Oriented: Traditional ciphers (A-Z).
  • Bit-Oriented: Modern ciphers (8 or 16 bits). Essential for encrypting non-text data like graphics, audio, and video.
  • Security: Bit-oriented ciphers increase the number of symbols in plaintext by 8 to 16 times, increasing security significantly.

7. Key Management System

  • Generation: Keys must be random and long enough to avoid prediction.
  • Distribution: Moving keys to the user; difficult in symmetric systems as it requires a secure channel.
  • Installation: Placing the key into device storage.
  • Storage: Preserving integrity/confidentiality (e.g., storing keys in encrypted form).
  • Change: Ending one key and starting another to lower interception risks.
  • Control: Regulating the use and selection of keys for specific applications.
